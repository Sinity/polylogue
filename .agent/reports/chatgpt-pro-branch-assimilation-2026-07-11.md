# ChatGPT Pro Branch Assimilation

Date: 2026-07-11

## Scope and provenance

This adjudicates the ten captured ChatGPT conversations titled
`Branch · Project Explanation and Relevance`. They are the divergent branches
represented by escrow folders 13 through 22 under
`.agent/scratch/gpt-fork-deliveries-2026-07-10/`. The shared conversation root
and the other twelve captures in that escrow are outside this report.

The source captures are the `6a511*.json` files under
`/realm/inbox/polylogue-browser-spool-2026-07-10/chatgpt/`. `INDEX.md` and
`VALIDATION.md` in the escrow establish capture identity and recovery status.
Per-branch `delivery.md`, `inline-artifacts.md`, and `STATUS.md` preserve the
recoverable content.

On 2026-07-11 the authenticated ChatGPT conversation API was probed directly
from a background CDP target. This disproved the earlier blanket assumption
that the sandbox packages had expired. Recovered bytes are preserved under
`/realm/inbox/gpt-pro-sol/recovered-branch-project-explanation-2026-07-11/`
with a SHA-256 manifest. The Query DSL and Sinex test-vacuity archives return
`ace_pod_expired`; the Cockpit v2 archive returns `404 Interpreter file not
found`, although its earlier Cockpit kit survives. All other branch packages
or source files needed below were recovered.

Rules used here:

- Lost sandbox patches, test output, manifests, and ZIPs are not implementation
  evidence.
- A report claim is checked against current source and Beads before adoption.
- Existing Beads remain planning authority; no parallel campaign graph is
  imported.
- `accepted` means useful current design or acceptance input, not proof that
  the report's implementation landed.
- `superseded` means current source or later verified work replaced the claim.
- `rejected` means the proposal should not be adopted.
- `regenerate` means reimplement a bounded current-source slice, not recreate
  the expired package byte for byte.

## Result

The ten branches are now classified, but recovery does not make them ten
apply-ready deliveries. Recovered code remains candidate code until rebased,
reviewed, and rerun against current source. Three Polylogue branches warrant
bounded implementation against current source (13, 15, 16); one warrants only
its residual constructs (17);
one warrants contract refinement through existing web children (14); one
requires a real missing persistence slice (20); one supplies launch strategy
with a necessary positioning correction (19). The three Sinex branches are
research inputs whose missing executable artifacts prevent direct graph or code
changes.

Durable Beads notes must cite this report and the corresponding escrow folder.
Any future implementation should cite the current owning bead, not the expired
ChatGPT sandbox package.
