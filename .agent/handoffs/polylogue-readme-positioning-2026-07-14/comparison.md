## Interpretation

- **Sinex recall-first** is the strongest repository front door: it reaches the shipped recall action quickly, shows a payoff before architecture, and exposes status/media boundaries. Its score does not validate the hand-built image.
- **Sinex replay-first** expresses the deepest technical differentiator. Use it for technical audiences or as the second screen.
- **Sinex rebuildable-first** is the concept/launch arm. Promote it only if readers retain the source-versus-interpretation model without losing the practical action.
- **Sinex event-spine** is conservative contributor positioning; it avoids prototype media but asks readers to care about implementation first.
- **Polylogue receipts-first** makes the differentiator memorable. Its no-install command remains a candidate until measured.
- **Polylogue search-first** reaches an action fastest but risks category collapse to “chat search.”
- **Polylogue continuity-first** is valuable for handoff material and must remain bounded to capability language.

## Promotion gate

Advance a candidate only when first-screen claims resolve to current code or a generated receipt, the clean-machine route succeeds and is timed, cold readers can state category/differentiator/boundary, and media is freshness-gated against a typed owner. Structural score alone is insufficient.
