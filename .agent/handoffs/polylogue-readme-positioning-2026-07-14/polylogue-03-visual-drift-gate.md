# Polylogue 03 — visual evidence drift gate

## Goal

Make the receipt image a generated evidence artifact whose recipe, typed output, tape, and committed media cannot drift independently.

## Current owners

- `polylogue/demo/receipts.py`;
- `tests/unit/cli/test_demo_command.py`;
- `devtools/visual_vhs.py`;
- `devtools/render_visual_tapes.py`;
- `docs/visual-evidence.md`;
- `docs/examples/visual-tapes/evidence-receipt.tape` and media.

## Contract

Add a compact typed presentation payload or use the existing JSON payload directly. The visual generator must not scrape arbitrary terminal text. Normalize only declared unstable fields such as temporary archive path; never normalize away the claim boundary, exit codes, refs, or anti-grep counts.

A strict check should:

1. generate the deterministic archive and canonical JSON;
2. validate claim, failed action, later recovery, anti-grep control, and source refs;
3. render compact text from typed data;
4. generate the tape from a declared spec;
5. render media in a pinned environment;
6. compare canonical JSON/text/tape/media with committed owners;
7. fail with a useful diff when any layer drifts.

## Seeded failures

Tests must fail when: the failed exit becomes zero; recovery disappears; recovery is moved before the claim; anti-grep failed-actions becomes nonzero; a visible ref is removed; prompt escape text reappears; the tape command changes without media regeneration; or committed media changes without its manifest/hash.

## Determinism boundary

Pixel-exact comparison is acceptable only under a pinned renderer/font/terminal stack. Otherwise compare canonical content plus a deterministic intermediate image and use a tightly bounded perceptual check for the final container. Record the environment identity either way.

## Acceptance criteria

`render ... --check` fails on every seeded drift; the generated image contains no raw private path; the README image is generated by this owner; and the visual manifest records command, versions, canonical payload hash, tape hash, media hash, dimensions, and non-claims.
