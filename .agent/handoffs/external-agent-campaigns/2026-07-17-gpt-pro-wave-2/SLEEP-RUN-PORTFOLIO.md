# Sleep-run portfolio — four executable support lanes

Attach one fresh Chisel archive from current `master` to every prompt. Launch
each in a separate chat and retain the Result-ZIP contract. This portfolio is
not a re-run of existing Wave 2 jobs: each support lane owns a reusable,
implementation-grade slice that its paired local lane does not own.

| Local lane | External support lane | Separate ownership boundary |
| --- | --- | --- |
| A: claim-vs-evidence + demos | `support-a` | first-party evidence objects and synthetic proof; local lane owns live/private calibration and report regeneration |
| B: Hermes forensics | `support-b` | verification-ledger importer and synthetic fidelity harness; local lane owns real `~/.hermes` byte validation |
| C: query/MCP closure | `support-c` | cross-surface transaction certification harness; local lane owns incident-scale archive proof and client rollout |
| D: raw-authority closure | `support-d` | deterministic crash/restart fixed-point harness; local lane owns live preflight and operator-authorized apply boundary |

These four can run immediately and in parallel. They are deliberately
independent of the `mandate-*` chain (`2qx.2` → `1vpm.6.2` → `z9gh.7`), which
should still run in its dependency order once each prior Result ZIP is
available. Do not launch already-obsolete prompts merely to fill capacity;
use fresh, distinct work with a named integration owner.
