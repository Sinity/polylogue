## Method and provenance

This census was built from the extracted project-state snapshot, its bundled Git repository, the full exported Beads JSONL, the generated MCP equivalence artifact, and source grep/semantic review. Snapshot archive SHA-256: `ec3fa193b2f99a6daee0bc620af4f69b5728834e99f8196792a17c3fbae11155`. Task file SHA-256: `a9a3b8975df03cb4937f4429f2dc53f7808a050a57856046fc5b89e19346f746`. Bundled HEAD: `536a53efac0cbe4a2473ad379e4db49ef3fce74d`.

The visible source tree matched the bundled Git clone at HEAD. The snapshot metadata marked the archive dirty and contained an extra `browser-extension/package-lock.json`; hidden project-state directories came from the Git bundle. Therefore path/line citations refer to the bundled source at that HEAD, while Bead citations refer to `polylogue-beads-export.jsonl` in the snapshot. No live browser/daemon/archive/deployment/test evidence is claimed.

The gap census is semantic: one row per surface entry or internal operation actuator that can change archive/user/derived/durable operation state. It intentionally excludes ordinary read-only SQL, in-memory mutation, formatting, temporary output files, auth-token maintenance, process PID/call-log/telemetry bookkeeping, and test-only writers. Those exclusions are not archive operation authority. Low-level storage helper calls are represented by their public semantic entry and by central choke points where the architecture decision requires special handling.

## Census conclusions

1. There is no production `OperationExecutor` call to inventory. The intended authority boundary is wholly absent, so every semantic mutation entry in the table is a bypass by construction (`polylogue/operations/specs.py:12-62`; bead `polylogue-t46.9`).
2. `ArchiveWriteGateway` cannot be promoted into that role unchanged. Its source contract is ingest commit/effect scheduling, and bead `polylogue-a7xr.18` confirms only ingest constructs it in production. It should become an effect/transaction dependency behind handlers.
3. MCP's present role registration is necessary but insufficient. `mcp_role_allows` is monotonic and includes `review`; authorization is declaration visibility, not per-operation executable proof (`polylogue/mcp/declarations/models.py:11-18`; `polylogue/mcp/server_tools.py:1424-1434`).
4. Current “delete session” paths are not semantically uniform. MCP/API/query/daemon paths prune rebuildable index rows; reset may additionally write suppression; excision destroys durable evidence across tiers. Keeping one ambiguous verb would make a correct bound-confirm protocol impossible (`polylogue/storage/sqlite/archive_tiers/archive.py:6328-6348`; `polylogue/cli/commands/reset.py:380-542`; `polylogue/security/excision.py:577-642`).
5. Judgment needs both layers: executor capability/CAS and an atomic user-tier transaction. Routing the existing race through an executor without fixing `upsert_assertion` would preserve a correctness bug (bead `polylogue-41ow`).
6. Internal system paths are not exempt. Startup repair, watched-query/candidate convergence, embedding, lifecycle excision, and ingest require explicit standing system capabilities and receipts through the same executor; the architecture says so directly (`ARCH-04:12-17,76-79`).
7. Live ingest currently performs a D2 system bypass: `_compact_superseded_raw_snapshots` invokes raw-retention deletion, which removes durable `raw_sessions`/`blob_refs` rows and then unlinks blob bytes (`polylogue/sources/live/batch.py:2256-2284`; `polylogue/storage/raw_retention.py:1146-1248`).
