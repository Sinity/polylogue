## Repository doctrine that constrains this implementation

`CLAUDE.md` / `AGENTS.md` establishes the key architecture rule: substrate/product layers own meaning; CLI, MCP, API, and daemon/Web surfaces are leaf adapters. This rules out a WebUI-local query schema, local filtering/ranking, or a second interpretation of continuation/coverage.

The same instructions require generated surfaces and focused verification. `devtools/generated_surfaces.py` is the repository mechanism that orders generated artifacts and gives `render all --check` its drift graph. `devtools/render_support.py` supplies deterministic write behavior. The new client is registered in that existing mechanism rather than adding a parallel CI script.

## Existing schema-generation evidence

### OpenAPI

`devtools/render_openapi.py` already builds `docs/openapi/search.yaml` from Pydantic models in `polylogue.surfaces.payloads`. Before this patch it declared six operations but had two contract defects relevant to generated clients:

1. `GET /api/sessions` always claimed `SearchEnvelope`, although the handler returns a different list envelope when `query` is absent.
2. `/api/query-units` declared `expression` required and did not declare its emitted continuation as an input.

The implementation publishes the existing `SessionListResponse`, fixes the response union, and adds paging metadata beside the operations.

### CLI output schemas

The existing CLI schema renderer demonstrates the repository's intended pattern: Python models are authoritative, generated artifacts are committed, and check mode detects drift. The WebUI generator mirrors that lifecycle rather than introducing an npm schema compiler or manually maintained interfaces.

## QueryTransaction evidence

`polylogue/archive/query/transaction.py` defines:

- `QueryTransactionRequest(operation, arguments, page_size, offset, projection, stable_order)`;
- logical `query_ref` independent of page offset;
- `QueryContinuation`, encoding the complete next request plus `result_ref` as an opaque `q1.` token;
- `QueryResultPage`, where continuation advances from the emitted row count.

`polylogue/archive/query/unit_results.py::query_unit_envelope()` already builds a canonical request with exactly:

```text
operation = query_units
arguments = {expression, session_filters}
page_size = request.limit
offset = request.offset
projection = default
stable_order = canonical
```

It derives stable `query_ref`, matching `result_ref`, and the next continuation. This proves the correct client behavior is to return the token unchanged, not to reconstruct filters or offsets.

## Production-route contradiction and repair

Before the patch, `polylogue/daemon/http.py::_handle_query_units()` always read `expression`, `limit`, `offset`, and filters directly from query parameters. It returned the token generated by `query_unit_envelope()`, but had no branch that called `QueryContinuation.decode()`.

Consequences before repair:

- the server advertised a continuation that could not advance its own route;
- a generated async iterator could only be cosmetic or client-reconstructed;
- the HTTP surface diverged from the shared `QueryTransaction` vocabulary required by `polylogue-z9gh.9.1`.

The repair consumes the token, rejects overrides, validates request identity, and rebuilds the existing `QueryUnitRequest`. No query semantics move into the HTTP layer; it remains an adapter over `query_unit_request()` and `query_unit_envelope()`.

The decoder also previously let some malformed base64/UTF-8 failures escape its documented `ValueError` boundary. Catching `binascii.Error` and `UnicodeDecodeError` makes malformed client input consistently map to the daemon's typed `invalid_continuation` response.

## Search cursor evidence

Ranked search already has a stable cursor/keyset implementation and `SearchEnvelope.next_cursor`. Its request still needs the original semantic filters, while `offset` is an initial-page concern that must not accompany cursor continuation. This is why the OpenAPI page declaration uses:

```text
continuation_request = merge
reset_parameters = [offset]
```

The generated iterator preserves query/provider/limit/retrieval filters, removes offset, and adds the opaque cursor.

`SearchEnvelope.exactness` distinguishes `exact`, `capped`, `sampled`, and `estimate`. This is direct evidence for the generic coverage union. Numeric `total` alone is insufficient to claim exactness.

## Error-envelope and same-origin evidence

The daemon's error helper emits the common shape:

```json
{
  "ok": false,
  "error": "machine_code",
  "detail": "human detail or null",
  "field": "field or null"
}
```

Web credential admission also reports request and credential-state headers. `FetchTransport` maps those existing fields; it does not invent a different error protocol.

Daemon Web routes are same-origin first-party surfaces. The runtime therefore permits only origin-relative generated paths, uses same-origin credentials, and rejects URL/path forms that could escape the configured origin.

## Contradictions and decisions

### Contradiction: session response declaration

**Observed:** OpenAPI described `/api/sessions` as `SearchEnvelope` even when the handler returned list mode.

**Decision:** publish existing `SessionListResponse` and use a response union. Do not coerce list mode into the search shape.

### Contradiction: continuation produced but not accepted

**Observed:** query-unit responses contained `q1.` continuations; the HTTP route did not decode them.

**Decision:** repair the route at the adapter boundary and validate the complete request identity. Do not reconstruct continuation state in TypeScript.

### Contradiction: mission says declaration kernel â€śwhen it landsâ€ť

**Observed:** declaration and MCP declaration packages exist in current source.

**Decision:** treat current source as authority. Describe retargeting as daemon/OpenAPI derivation migration, not creation of the already-present kernel.

### Contradiction: continuation-only WebUI list requirement vs current server

**Observed:** ranked search has a cursor; plain list remains offset-based.

**Decision:** generate the honest union operation and ranked iterator only. Record webui-02 list continuation as a server contract gap.

### Decision: operation extension rather than client convention

OpenAPI cannot infer whether a string named `cursor` is complete request state, a ranked keyset handle, or a retry hint. The page extension is generated server metadata colocated with the operation. It is narrower and safer than route-name conditionals in the TypeScript generator.

### Decision: fail closed on request bodies

Current declared operations require no request body. Silently ignoring a future body would create a convincing but unusable client. Generation therefore fails until body support is deliberately added.
