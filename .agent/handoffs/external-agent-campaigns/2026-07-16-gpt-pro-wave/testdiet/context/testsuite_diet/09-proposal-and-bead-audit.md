# Proposal and Bead audit

This audit changes the execution interpretation of earlier proposals. It does
not close or rewrite Beads from scratch research. Tracker owners should carry
the corrections into a Bead before implementation when the Bead text still
contains the rejected shape.

## Proposal shapes removed from this plan

- No coverage-domain catalog, quality score, naming allowlist/denylist, or
  “every test mapped” gate.
- No test that reads source merely to require a new helper/module/table spelling
  or forbid the old one.
- No independent oracle implemented by translating production query/storage
  rules into a second hand-authored shadow model.
- No MCP or current web-reader cleanup wave beyond externally urgent safety or
  explicitly rewrite-owned contract extraction.
- No suite-size projection from census hits, mocked-call counts, coverage
  overlap, or dossiers. Only merged cluster economics enter the ledger.
