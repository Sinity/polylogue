## Preserve the name distinction

`devtools/dev_loop.py` is live branch-local daemon/web/browser-capture tooling.
Its process cleanup, port allocation, receiver polling, browser smokes, daemon
launch, artifacts, docs, and tests are unrelated to the retired task conductor.
Keep it and audit individual tests only against its operational contracts.

## Closed-loop deletion/simplification

- `devtools/verify_closure_matrix.py`: path/list/prose consistency only; no
  representative behavior executes.
- `devtools/scenario_coverage.py`: joins authored scenario targets to an
  authored artifact/operation graph. Preserve explicitly named navigation if
  useful; remove “coverage,” completeness, and strict-proof claims.
- `devtools/render_quality_reference.py`: keep generated navigation over real
  executable lane/mutation/benchmark registries. Remove runtime-coverage
  scorecards, scenario-projection-as-evidence language, and closure guidance.
- `devtools/verify_manifests.py`: remove authored coverage-gap schemas,
  path-existence coverage claims, implemented/test-location consistency,
  test-domain mirrors, campaign YAML mirrors, and command-substring CI claims.
  Preserve mutation/benchmark artifact existence, non-emptiness, and freshness
  as direct campaign receipt checks. Retain generic schema validation only for
  manifests with an independent purpose.
- `devtools/verify_docs_coverage.py`: keep. It derives live
  CLI/MCP/config/route inventories and ratchets documentation debt.
