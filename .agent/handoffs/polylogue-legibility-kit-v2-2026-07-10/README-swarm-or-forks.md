# Sixteen parallel fork missions

These prompts are designed to be launched as forks of the same repository-analysis conversation. The mission plan controls dependencies and scarce resources; do not start all implementation lanes simultaneously merely because all prompts exist.

1. [Mission 01 — Make “show me the receipt” executable](01-receipts-and-claims.md)
2. [Mission 02 — Make incomplete evidence impossible to mistake for complete evidence](02-readiness-and-degradation.md)
3. [Mission 03 — Turn public truthfulness into a release gate](03-claims-ci.md)
4. [Mission 04 — Make Polylogue understandable before the reader scrolls](04-readme-and-antigrep.md)
5. [Mission 05 — Make the generated site evidence-first](05-site-and-landing.md)
6. [Mission 06 — Produce proof media that can be inspected, not merely admired](06-proof-media.md)
7. [Mission 07 — Attack the public story as a hostile but fair reader](07-cold-reader-audit.md)
8. [Mission 08 — Make transcripts read like work, not serialized provider payloads](08-semantic-renderer.md)
9. [Mission 09 — Graduate Receipts from contract proof to one real merged change](09-real-pr-receipts.md)
10. [Mission 10 — Prove “store every artifact, count copied work once”](10-count-it-once.md)
11. [Mission 11 — Publish findings without laundering them into product claims](11-findings-shelf.md)
12. [Mission 12 — Prove the first experience from a clean environment](12-install-and-release.md)
13. [Mission 13 — Make uncertainty visible in the web reader](13-web-degraded-states.md)
14. [Mission 14 — Give Sinex two falsifiable proofs of its distinctive semantics](14-sinex-proof-demos.md)
15. [Mission 15 — Make the maximal Sinex-backed Polylogue design executable](15-maximal-interop.md)
16. [Mission 16 — Conduct the swarm and publish only what survives integration](16-integration-conductor.md)

The integration conductor is the only prompt authorized to run broad release verification or publish the combined result.
