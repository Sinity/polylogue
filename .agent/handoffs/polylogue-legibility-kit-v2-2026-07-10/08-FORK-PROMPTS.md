# Sixteen fork prompts

The prompts under [fork-prompts/](fork-prompts/README.md) are written to be pasted into parallel forks of the current conversation. Each fork receives the shared repository-analysis context and a bounded mission.

The primary set is:

1. Receipts and claims gate.
2. Readiness and degradation.
3. Claims CI.
4. README and anti-grep.
5. Site and landing.
6. Proof media.
7. Cold-reader attack.
8. Semantic renderer.
9. Real merged-PR Receipts.
10. Count It Once.
11. Findings shelf.
12. Install and release.
13. Web degraded states.
14. Sinex proof demos.
15. Maximal interop.
16. Integration conductor.

Do not launch all sixteen implementation missions simultaneously. Use `swarmctl.py ready` and the resource schedule. Static audit and documentation forks can run while scarce Python-heavy, Rust-heavy, browser, live-stack, and verification lanes are serialized.
