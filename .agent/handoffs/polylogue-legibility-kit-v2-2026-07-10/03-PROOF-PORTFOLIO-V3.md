# Proof portfolio v3: demonstrations that can lose

The goal is not to make the systems look capable under favorable conditions. It is to expose the distinctions that make them valuable and give each demonstration a clear way to fail.

## The proof ladder

A single “demo” label hides different epistemic jobs. The portfolio should use four explicit levels.

### Level 1 — Contract proof

A deterministic, private-data-free corpus exercises a narrow product contract through normal code paths. It proves implementation behavior under declared fixtures.

Examples: Receipts Contract Proof; Count It Once; Import It Twice.

### Level 2 — Field proof

A real historical case is reconstructed against an independent source such as Git objects, CI, source continuity, or immutable provider material. It proves that the product can recover one real case under a declared population.

Examples: one merged-PR Receipts packet; one production capture outage.

### Level 3 — Negative proof

The product is given insufficient, stale, ambiguous, or missing evidence and must refuse a stronger conclusion.

Examples: browser source unobserved; ambiguous duplicate; unsupported provider field; stale context pack.

### Level 4 — Comparative experiment

Two or more interventions are compared under frozen checkpoints, independent scoring, and a publication rule that includes losses.

Examples: Resume Under Oath; lexical versus hybrid retrieval; reviewed context versus generated summary.

No level substitutes for another.

## Shared proof world: Incident 14:32

The shared corpus exists to reduce narrative overhead while preserving independent oracles. It contains:

- a focused test failure before an assistant success claim;
- a later edit and passing rerun;
- transcript and independent terminal agreement;
- a prose-only anti-grep control;
- a parent, copied-prefix fork, and fresh subagent;
- declared physical and logical token totals;
- a browser interval known to be unobserved;
- a Bead closed by a named commit;
- a rejected assistant assertion and accepted operator lesson;
- a delivered context image with explicit omissions;
- parser semantics v1 and v2 sharing stable domain identity but not interpretation event identity.

The corpus is deliberately not a product export. Its verifier hashes all declared materials and derives 24 facts independently.

## Public launch arc

### 1. Polylogue — The Receipts: contract proof

**Question:** Did the structural evidence support the assistant’s claim at the moment it was made?

**Construct:** claim-versus-tool-result comparison at a temporal boundary.

**Oracle:** normalized exit code and error state anchored to source material.

**Controls:**

- prose contains “error” three times but has zero actions;
- a later successful test proves recovery without rewriting the earlier boundary.

**Falsifier:** changing only prose wording changes the verdict, or changing exit status does not.

**Receipts:** message ref, tool result ref, source material/blob hash, timestamps, later repair ref.

**Non-claims:** provider completeness, general model error rate, field prevalence, causality.

**Current state:** implemented in the supplied Polylogue patch.

### 2. Polylogue — The Receipts: field proof

**Question:** Can the same method reconstruct one real merged agent-authored change?

**Construct:** transcript claim and action evidence joined to independently frozen Git/CI outcome.

**Oracle:** Git objects, merged change metadata, and test/CI evidence not authored by the transcript renderer.

**Controls:** an unmerged attempt or pre-verification claim if naturally present.

**Falsifier:** the packet’s conclusion depends on PR prose or cannot resolve the actual change and verification.

**Current state:** planned under `polylogue-212.2`; do not claim it from the deterministic proof.

### 3. Polylogue — Count It Once

**Question:** Can the archive preserve every physical artifact without charging copied session prefixes as new work?

**Construct:** physical versus logical lineage accounting.

**Oracle:** declared stable message identity graph and token lanes.

**Controls:**

- fresh subagent remains unique;
- unrelated identical text remains unique;
- ambiguous lineage yields a caveat.

**Falsifier:** physical evidence disappears, or fresh work is subtracted.

### 4. Sinex — The Missing Source

**Question:** Is an empty interval actually observed?

**Construct:** source coverage in the query result.

**Oracle:** independent source-health and continuity records.

**Controls:** known-observed empty interval and known-unobserved interval.

**Falsifier:** both return the same ordinary empty state.

### 5. Sinex — The System Changes Its Mind Honestly

**Question:** Can a new interpretation replace the current view without deleting the old reading?

**Construct:** occurrence identity versus interpretation identity, replay, archive, and promotion.

**Oracle:** two frozen parser outputs over identical source material.

**Controls:** importing the same occurrence twice; replaying under changed semantics.

**Falsifier:** repeated acquisition duplicates the occurrence, old interpretation vanishes, or promotion is implicit.

### 6. Joint — The World Around the Claim

**Question:** What did the agent say, what did its tool report, what did the machine do, what changed in Git, what task was intended, and which sources were missing?

**Construct:** cross-domain Agent Work Packet retaining native authorities.

**Oracle:** Incident 14:32 timeline and material hashes.

**Controls:** missing browser leg and pre-repair success claim.

**Falsifier:** the packet invents a missing leg, collapses task intent into transcript prose, or treats a Git commit as proof that the test already passed at claim time.

### 7. Joint — Resume Under Oath

**Question:** Does reviewed evidence-linked context improve state reconstruction without amplifying stale claims?

**Arms:**

1. raw references;
2. generated summary;
3. reviewed context image.

**Oracle:** frozen checkpoint answers and blind scoring.

**Controls:** stale assertion trap, omitted-evidence checkpoint, equal token and effort budgets.

**Falsifier:** reviewed context loses, increases high-confidence falsehoods, or depends on evaluator knowledge of arm labels.

The publication rule must include losses. “Directional benefit” is not a general uplift claim.

## Longer-horizon proof portfolio

### Context Autopsy

Reconstruct exactly what crossed the model boundary, what was omitted, and which assertion policies applied. Compare agent behavior with evidence unavailable versus present.

### Compaction Autopsy

Freeze pre-compaction context, provider summary, post-compaction context, and later failure. Test whether the lost fact was actually absent rather than narratively blamed after the event.

### Deletion Under Lineage

Delete one transcript/material set with shared-prefix relationships. Prove domain-aware reference accounting, projection invalidation, FTS/vector/context purge, and absence of over-cascade.

### Crash at the Receipt Barrier

Fault-inject process death between material staging, event admission, settlement, projection, and acknowledgment. Prove either replayable recovery or explicit unresolved debt.

### Retrieval That Earns Its Cost

Evaluate lexical, semantic, and hybrid retrieval on frozen query/relevance judgments. Record model effect recipe, dimensions, chunking, cost, latency, and hit@k. Do not use embeddings merely because they exist.

### Privacy Disclosure Control

Show the same object under generic Sinex MCP, privileged Polylogue service, and exported public proof contexts. Prove that raw content capability and metadata/search capability are distinct.

## Demo ideas rejected as first-contact proofs

### Infrastructure tour

Showing PostgreSQL, SQLite tiers, NATS, or daemon stages proves implementation effort, not product value. Retain it for operators after a user-facing proof.

### Capability montage

A rapid sequence of search, costs, embeddings, context, MCP, and dashboards creates breadth but no falsifiable construct. Use a portfolio index after individual proofs exist.

### Private archive scale as hero claim

Millions of messages or billions of tokens demonstrate dogfooding, but they are difficult for outsiders to reproduce and encourage performance assumptions. Publish as a bounded field observation, not the first action.

### “Better than grep” comparison table without an executable control

A table reads like marketing unless the anti-grep fixture demonstrates that prose keywords and structural outcomes diverge.

### Live autonomous multi-agent spectacle

A swarm can look impressive while hiding failures, hand-curated prompts, and human intervention. First prove mission leasing, context delivery, evidence refs, and outcome packets; then run the live swarm with a full event log.

### Agent leaderboard

Model/provider comparisons invite confounding from task selection, context, tools, and archive coverage. Build method and checkpoint controls first.

### Generic timeline

A timeline is a presentation primitive, not a thesis. It becomes compelling when it distinguishes observed empty, missing source, revised interpretation, or conflicting evidence.

### Memory demo judged by eloquence

A polished continuation is not proof of correct state reconstruction. Use frozen answers, blind scoring, stale traps, and equal budgets.

## Demo packet requirements

Every public packet must contain:

- stable demo ID and version;
- evidence class;
- primary construct;
- predeclared claim;
- independent oracle and why it is independent;
- positive, negative, and missing-evidence controls where applicable;
- falsifier;
- exact commands and environment;
- material manifest and hashes;
- machine-readable result;
- human presentation;
- refs to underlying evidence;
- scope and population;
- non-claims;
- regeneration and interruption behavior;
- owning Beads;
- supersession status.

A screenshot without this packet is media, not proof.
