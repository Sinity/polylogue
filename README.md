Polylogue is a local archive and analysis layer for AI conversations. It turns
ChatGPT, Claude, Claude Code, Codex, and Gemini exports or captures into one
SQLite archive with full-text search, materialized session products, cost
estimates, and static publication output.

**Local-first.** Everything lives under your XDG data directory. No cloud
dependency.

## Quickstart

### Nix (try without installing)

```bash
nix run github:Sinity/polylogue -- --help
```

### From source

```bash
git clone https://github.com/sinity/polylogue
cd polylogue
direnv allow   # or: nix develop
```

### Try it with synthetic data from a source checkout

Synthetic demo data is generated by `devtools`, the verification-lab and
repository-control-plane CLI. The product CLI is `polylogue`; `devtools` is
shipped and documented so source checkouts, CI, and packaged developer
installs can reproduce demos and proof artifacts with the same command surface.

```bash
eval "$(devtools lab-corpus seed --count 8 --env-only)"

polylogue list --limit 5
polylogue products profiles --limit 5
polylogue run site -o ./site-preview
```

Demo data is isolated from your normal archive. If you only installed the
product CLI and do not have `devtools`, skip this section and ingest a real
export with `polylogue run --input`.

## Ingest real exports

Download a ChatGPT or Claude export, then:

```bash
polylogue run --input ~/Downloads/conversations.json
polylogue run --input ~/Downloads/chatgpt.zip
```

The `--input` flag accepts files, directories, and zips. Duplicate imports are
silently idempotent — content hashes prevent re-import of the same data.

**Configured sources** (repeating paths like `~/.claude/projects/`) differ from
one-shot inputs. Add them with `--source`:

```bash
polylogue run --source claude-code --source codex
```

Sources are discovered from your local environment automatically. Use
`polylogue run --preview` to see what work a run would perform before executing.

## CLI shape

The root command is query-first — bare tokens are search queries:

```bash
polylogue "error handling" list --limit 10
polylogue --provider claude-code --since "last week" stats --by provider
polylogue --provider claude-code --repo __thoughtspace list
polylogue --has-tool-use --typed-only list --limit 5
polylogue --latest open
```

Pipeline and maintenance verbs are explicit:

```bash
polylogue run acquire parse materialize render index
polylogue doctor --daemon
polylogued status
polylogued watch
polylogued browser-capture serve --host 127.0.0.1 --port 8765
polylogue-mcp --role read
```

See [docs/cli-reference.md](docs/cli-reference.md) for the full command reference.

## Core surfaces

### Archive and query

- Provider detection from file content (not filenames)
- Full-text search with composable filters (provider, date, tag, repo, tool, action)
- Precomputed message analytics: tool use, thinking blocks, paste detection
- Query-time tail overlays for sessions newer than the archive

### Durable products

```bash
polylogue products profiles --limit 10
polylogue products phases --limit 10
polylogue products threads --limit 10
polylogue products costs --limit 10
```

Products are materialized from the archive and updated incrementally.

### Browser capture

```bash
polylogued browser-capture serve --host 127.0.0.1 --port 8765
polylogued browser-capture status
polylogued status
```

Captures ChatGPT and Claude.ai browser sessions through a local receiver and
extension. The unpacked extension source lives in `browser-extension/`.

### Publication and MCP

```bash
polylogue run site -o ./site-preview
polylogue-mcp --role read
```

- Static HTML archive with search
- MCP stdio bridge for AI assistant integration

### Python API

```python
from polylogue import Polylogue

async with Polylogue() as archive:
    convs = await (
        archive.filter()
        .provider("claude-code")
        .contains("error handling")
        .limit(10)
        .list()
    )
```

The Python API is async-first. See [docs/library-api.md](docs/library-api.md).

## Developer tools

devtools verify

## Verification lab

```bash
devtools render-verification-catalog --check
devtools affected-obligations --path polylogue/cli/query.py
devtools lab-corpus representative-generate -p chatgpt -n 3
```

The verification lab records proof-obligation subjects, claim runners, and
evidence routing so agents can choose focused checks before escalating to the
full repository baseline.
