Reader-comprehension test runner (polylogue-3tl.19)

Single-blind, N-arm cold-reader test for a README/positioning candidate. Randomize an arm, expose it for a fixed duration, hide it, capture unaided answers, then re-show for the first-action and false-belief questions. Sessions persist in this browser only (localStorage) until exported. This tool measures comprehension of a fixed visual exposure, not the full live README experience (scrolling, following links, running commands) — see the companion doc's non-claims section before treating a result as decision-grade.

Arm configuration

Point each arm at a real, already-generated screenshot (a full-page capture of the candidate README's first fold, not a hand-drawn mockup — see docs/examples/reader-comprehension-test/README.md for how to produce one). An arm with no configured image shows an explicit placeholder instead of a broken <img>, so a missing screenshot cannot be silently scored as "read fine."

Independent scoring

Scored by someone other than the participant, against the rubric in the companion doc (not against the scorer's own opinion of the copy). Each dimension anchors to one of docs/demos.md's four doctrine terms: category/outcome/differentiator = claim recognition; boundary = whether the reader noticed the material non-claim; false beliefs = the falsifier (any hit here means the arm failed regardless of the numeric scores).

Non-claims (read before running a real study)

  • A handful of sessions from this tool is not a population-prevalence claim about how README readers in general respond — sample size and selection must be stated alongside any result (polylogue-3tl.19 AC4).
  • This tool does not measure downstream agent performance or task success; it measures stated comprehension and recall after a timed visual exposure.
  • A candidate advances only when it wins or ties comprehension without introducing a new false belief — a higher category/outcome score with a nonzero false-belief count is not a win.
